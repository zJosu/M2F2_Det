# Plan: Fine-tuning M2F2-Det for AI-Generated Image Detection

## Objective
Adapt the M2F2-Det model (originally trained on FaceForensics++ face manipulations) to detect **fully AI-generated images** from diffusion models (Grok/xAI, Stable Diffusion, DALL-E, Midjourney, etc.).

---

## Current Model Limitations

The current M2F2-Det model was trained on **FaceForensics++**, which includes:
- **Deepfakes** (face swap via autoencoders)
- **FaceSwap** (3D model-based face swap)
- **Face2Face** (facial reenactment)
- **NeuralTextures** (neural rendering)

These are all **face manipulation** techniques that modify **parts of a real image**. Fully AI-generated images from diffusion/transformer models have fundamentally different artifact patterns, so the model fails to detect them.

---

## Architecture Overview

### What gets fine-tuned at each stage:

| Stage | Component | Trainable | Purpose |
|-------|-----------|-----------|---------|
| **Stage 1** | M2F2Det (DenseNet121 + CLIP + Bridge) | All (except frozen CLIP) | Binary classifier: real vs fake |
| **Stage 2** | deepfake_projector (MLP 2→4096→4096) | Only projector | Align deepfake features → LLM space |
| **Stage 3** | LoRA adapters + projectors | LoRA on LLM + projectors | Full detection + explanation |

### Key: The LLM-integrated model uses **DenseNet121** (not EfficientNet-B4 from standalone Stage 1)

The checkpoint `M2F2_Det_densenet121.pth` feeds into `LlavaLlamaForCausalLMDeepfake`.

## Phase 2: Stage 1 Fine-tuning (Binary Classifier)

### Script: `finetune_aigen/stage_1_finetune.py`

**Architecture**: M2F2Det with DenseNet121 (matching LLM-integrated version)

**Strategy**: 
- **Warm start**: Load current `M2F2_Det_densenet121.pth` weights
- **Freeze CLIP encoders** (already pretrained)  
- **Fine-tune**: DenseNet121 backbone + bridge adapter + projections + output head
- Low learning rate (1e-4) to preserve existing knowledge while learning new patterns

**Hyperparameters for Colab T4**:
| Parameter | Value |
|-----------|-------|
| Batch size | 16 (gradient accumulation to 64) |
| Learning rate | 1e-4 |
| Epochs | 30-50 |
| Optimizer | AdamW |
| Scheduler | CosineAnnealingWarmRestarts |
| Image size | 224×224 |
| Precision | fp16 (mixed) |

**Data Augmentation**:
- Random horizontal flip
- Random JPEG compression (quality 30-100)
- Gaussian blur (sigma 0-3)
- Random crop + resize to 224×224
- Color jitter (brightness, contrast, saturation)

**Output**: Updated `M2F2_Det_densenet121_aigen.pth`

### Training on Colab T4:
- Memory: DenseNet121 ~30MB + CLIP ViT-L ~1.2GB ≈ ~2GB model
- Batch 16 × 224×224 ≈ ~2GB activations
- Total ≈ ~6GB → fits T4 (16GB) with room for gradients

## Phase 3: Create DDVQA-Format Data for LLM Stages

### Script: `finetune_aigen/generate_ddvqa_data.py`

**3.1 Judge-Only Data** (Stage 2: `train_DDVQA_aigen_judge_only.json`)
```json
{
    "id": "real_ffhq_00001",
    "image": "real_ffhq_00001.jpg",
    "conversations": [
        {"from": "human", "value": "<image>\n<deepfake>\nDo the image looks real or fake?"},
        {"from": "gpt", "value": "The image looks real."}
    ]
}
```

**3.2 Full Conversation Data** (Stage 3: `train_DDVQA_aigen_full.json`)
```json
{
    "id": "aigen_sd_00001",
    "image": "aigen_sd_00001.jpg",
    "conversations": [
        {"from": "human", "value": "<image>\n<deepfake>\nDescribe the authenticity of the image."},
        {"from": "gpt", "value": "This image appears to be fully generated by an AI model. The skin texture is unnaturally smooth and uniform. The lighting and shadows show subtle inconsistencies typical of diffusion-model outputs."},
        {"from": "human", "value": "Is the image real or fake?"},
        {"from": "gpt", "value": "Based on the learned representation, this image is fake."}
    ]
}
```

**Explanation templates** for AI-generated images:
- Overly smooth/perfect skin texture
- Inconsistent lighting/shadows
- Background inconsistencies
- Unnatural hair/ear details
- Too-perfect symmetry
- Characteristic diffusion model artifacts

## Phase 4: Stage 2 Fine-tuning (Projector Alignment)

### Script: `finetune_aigen/stage_2_train.sh`

**What's trained**: Only `deepfake_projector` (frozen LLM + frozen encoder)

On Colab T4 with 4-bit quantization:
| Parameter | Value |
|-----------|-------|
| Epochs | 1-2 |
| Batch size | 4 (gradient acc. to 16) |
| Learning rate | 2e-5 |
| Precision | fp16 |

This stage is quick (~30 min on T4 with small dataset).

---

## Phase 5: Stage 3 Fine-tuning (LoRA + Full Conversation)

### Script: `finetune_aigen/stage_3_train.sh`

**What's trained**: LoRA adapters on LLM + deepfake_projector + mm_projector

| Parameter | Value |
|-----------|-------|
| Epochs | 1-2 |
| Batch size | 2 (gradient acc. to 8) |
| Learning rate | 2e-5 |
| LoRA rank | 64 (reduced from 128 for T4) |
| LoRA alpha | 128 |
| Precision | fp16 + 4-bit base |

---

## Phase 6: Evaluation

### Script: `finetune_aigen/evaluate.py`

- **Metrics**: Accuracy, AUC-ROC, F1, Precision, Recall
- **Per-generator breakdown**: Performance on each AI generator
- **Cross-generator generalization**: Test on unseen generators
- **Comparison**: Before vs after fine-tuning

## Execution Order

```
1. prepare_dataset.py          → Download/organize dataset
2. stage_1_finetune.py         → Fine-tune DenseNet121 encoder  
3. generate_ddvqa_data.py      → Create conversation training data
4. Stage 2 training            → Align projector with new features
5. Stage 3 training            → LoRA fine-tune LLM
6. evaluate.py                 → Test on held-out data
```
